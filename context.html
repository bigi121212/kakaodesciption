<컨텍스트>

사용자와 채팅 봇 사이에 하나의 주제를 가지고 대화할 때 쓴다.
블럭과 블럭을 이어주는 다리 역할이라고 생각하면 쉽다.

구체적으로 말하면, 문맥에 맞는 대화를 챗봇과 하기 위한 것이다.
무조건 한 방향으로만 진행하는 일방통행이다.
컨텍스트 설정 시 다리를 빠져 나올 수 있는 조건을 걸 수 있다.

블록1과 블록2가 있고 컨텍스트로 묶는다고 생각해보자.
내가 원하는 대화의 진행은 블록1을 시작으로 블록2로 나아가고자 한다

블록1은 블록2로 출력하는 역할이기에 Output컨텍스트를 설정해야 한다.
반대로 블록2는 블록1로부터 대화의 진행을 받아들이는 입장이기에 입력을 받는 Input컨텍스트 설정.
*중요한 것은 Output과 Input 모두 같은 이름의 컨텍스트로 묶어 주어야 한다.

Output 컨텍스트의 설정에 따른 채팅 결과

1회 10분 가정

  1. 블록1을 실행하는 발화를 사용자로부터 입력 받으면
    컨텍스트 이름=블록1, 남은횟수=1 (테스트 채팅창을 키면 볼 수 있다) 과 같이 뜬다
    이 때, 블록2의 발화를 사용자로부터 입력받으면 대화가 이어진다.
    즉, 블록1에서 블록2로 대화가 진행함.

  2. 블록1의 발화를 사용자로부터 받고 전혀 상관없는 블록3를 실행
    결과: 블록3의 결과물을 출력함
    즉, 블록1의 대화를 시작했지만, 대화가 끊기고 새로운 대화 블록3 진행.

  3. 블록1의 발화를 사용자로부터 입력 받지 않고 블록2 실행
    결과물로 폴백블록이 실행.
    즉, 블록1의 입력없이는 블록2는 절대 실행 불가하다

*블록1에서 블록2로 파라미터 정보의 전달이 가능하다 그러나 사용자의 발화 정보를 보낼 수 없다.
